Even ARM proccessor support is located in all current kernel versions, to find out last changes visit ARM Linux Homepage. Our patch is currently based on patch-2.4.19-rmk7.bz2 WEP platform is based on Intel XScale ARM processor, so also visit Monta Vista kernel for diff-2.4.19-rmk7-pxa1.gz file. Patches created by ARM crew, namely Mr. Russel King as coordinator, are named patch-2.4.19-rmkX. The X is number of patch release (greater is newer) to vanilla kernel from www.kernel.org. Monta Vista kernel patches are named diff-2.4.19-rmkX-pxaY, so X defines RMK patch this diff refers to and Y is version number of current pxa patch. Actually our patch is naming conventions uses the same notation - patch-2.4.19-rmkX-pxaY-wepZ.

To get real patched sources ready for WEP platform, download vanilla kernel, use patch from ARM Linux of MR.King, use pxa patch from MontaVista and finally use our special WEP patch.

Our current patch is patch-2.4.19-rmk7-pxa1-wep2 and has these features ready
  • bootable on JFFS2 filesystem
  • support for 3 Hitachi displays in framebuffer 8bit and 16bit
  • touchpad support (tested on Xfbdev XFree server)
  • lcd control driver (supported by our utility)

WEP platform has some of the HW equipment already supported by ARM version of kernel, but not tested by our development team, so we do not recommend to rely to be working properly. Feel free to report such a functionality or submit your own patches.

Building instructions

Place all files into one directory and uncompress linux-2.4.19.tar.bz2 with 
  tar xjf linux-2.4.19.tar.bz2
enter linux directory 
  cd linux-2.4.19
and apply all needed patches 
  bunzip2 -c ../patch-2.4.19-rmk7.bz2 | patch -p1
  gunzip -c ../diff-2.4.19-rmk7-pxa1.gz | patch -p1
  gunzip -c ../patch-2.4.19-rmk7-pxa1-wep2.gz | patch -p1
To get apply WEP kernel configuration to your patched sources type 
  make wepep250_config
Optionally you may have to change some setups in kernel by hand. For example, this config is setup as monolitic kernel, so no kernel modules are compiled. Also setting framebuffer to type of your display is neccessary. Do 
  make menuconfig
If sources are properly setup and you have arm-linux-gcc named cross compiler available you may type 
  make dep;make
Hopefully a result vmlinux.bin file is placed in the direcory. For use with U-Boot loader you need to create U-Boot image file type like this 
  mkimage -A arm -O linux -T kernel -a a0008000 -e a0008000 -C none vmlinux.bin uImage
"mkimage" is tool compliled with U-Boot compilation. More intructions about loading image into U-Boot see our U-Boot help page.
